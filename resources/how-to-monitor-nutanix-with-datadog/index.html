How to monitor Nutanix with Datadog.

The recommended approach, the common pitfalls, and the teams this guide is for.

Three rules that save quarters.

Use a typed collector

Pull from Prism API v3 plus Redfish plus IPFIX rather than generic exporters. Keep metric semantics intact.

Filter at the source

Forward enriched metrics, not raw events. Cloud analytics cost stays predictable.

Keep evidence local

Store the full-fidelity copy on-prem. Forward summaries to Datadog for SRE visibility.

Three mistakes to avoid.

Scraping Prism UI

UI scraping breaks on upgrade. Always go through the typed API.

Sending raw flow records

Datadog ingest costs scale poorly with raw flows. Send aggregates plus anomalies.

Skipping hardware telemetry

Generic Datadog agents miss Redfish and IPMI. Hardware degradation surfaces too late.

If this is your team, it is your guide.

Teams already on Datadog

Existing SLOs, on-call rotations, and runbooks: keep them. Nutanix telemetry just appears in the right places.

Multi-cluster operators

One Datadog dashboard across every Nutanix cluster, without one Datadog Agent per cluster.
